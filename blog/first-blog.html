AI Agent 辅助搭建个人主页实践与思考

发布时间：2026-05-07 · 主题：AI Agent 与个人主页实践

前言

这是算法基础课程大作业的实践记录。我全程借助Codex 接入 GPT Plus完成个人主页搭建，从完全陌生到逐步上手，既体会到 AI Agent 带来的效率提升，也遇到了不少真实问题。本文把完整过程、踩坑经历与思考整理出来，既是作业交付，也是一次学习总结。

一、我用到的工具

  • AI 工具：Codex（接入 GPT Plus）
  • 页面编写：HTML + CSS
  • 部署上线：GitHub Pages
  • 文件存放：GitHub 仓库、GitHub Release、科大云盘
  • 项目规范：AGENTS.md

二、初次使用 AI Agent：从 “试探” 到 “协作”

刚开始接触 AI Agent 时，我完全不清楚它的能力边界。 不知道它能独立完成哪些任务、需要我提供哪些信息。 每次下达指令前都会先试探：“你能不能帮我……”“如果我想……需要我做什么？”

甚至连项目规范文档 AGENTS.md，都是我把作业要求与习题课讲义交给 AI，让它直接生成初稿。 整个过程更像 “一边用一边学”：我一边搭建主页，一边向 AI 学习如何正确使用它。这种边做边探索的模式，虽然起步慢，但很适合新手入门。

三、指令难题：模糊与过度精确之间的平衡

在实际使用中，我遇到了最核心的问题：指令尺度不好把握。

指令太模糊，结果就容易不符合预期。比如让 AI 设计主页分区，因为没有明确结构、风格、板块，生成的布局很乱，最后还是要我自己重新规划。

指令太精细，又会退化成普通对话。微调样式、改细节时，我把需求拆得特别碎，一步步问 AI，和普通对话框没区别，完全没发挥命令行 AI Agent 的优势，效率甚至不如我直接改源码。

这次实践让我明白：AI Agent 需要清晰但不过度细碎的指令，既要给目标，也要给边界。

四、大文件上传方案：分级存储策略

我的主页需要存放课程资料、小论文、PPT、视频等，文件大小差距极大，从几 KB 到几 GB 都有。我和 GPT 反复讨论后，确定了一套分级存储方案：

  • 小文件（≤30M）：论文、PDF、图片直接放入 GitHub Pages 仓库，访问最快、最稳定。
  • 中等文件（30M~几百 M）：PPT、课件包上传到 GitHub Release，通过链接引用，不占主页仓库体积。
  • 大文件（几个 G 视频）：没有找到合适的免费公开方案，最终上传到学校云盘，主页只放置跳转链接。

五、实践收获与思考

AI Agent 是 “协作伙伴”，不是 “替代者”。它能快速生成代码、写文档、给方案，但需求理解、结构设计、最终决策必须由人完成。

用好 AI 的关键是：会提问、会验收。模糊指令没用，过度指令浪费效率。学会把任务拆成目标明确、步骤清晰的任务，才能发挥 AI 最大价值。

AI 能提升效率，但不能代替思考与动手。页面报错、样式错乱、链接失效，最终还是要自己排查。AI 可以给思路，但解决问题仍靠自己。

AI Agent 编程会成为主流技能。这次作业让我体会到，未来学习算法、开发项目，AI 辅助会越来越普遍。提前学会和 AI 协作，本身就是一种重要能力。

总结

通过 AI Agent 搭建个人主页，我不仅完成了课程作业，更重要的是第一次真正体验了 AI 辅助开发的完整流程。从试探使用、优化指令，到解决文件存储问题，每一步都有真实收获。

AI 让入门更简单，但理解与思考依然是核心。未来我会继续用 AI 辅助学习算法与开发，让技术工具成为自己的助力。
